二、代理平台：Managed Agents 四大能力升级

::: summary 核心更新：Dreaming（梦境）、多代理编排、Outcomes（自评循环）、自托管沙箱 + MCP 隧道

2.1 Dreaming（梦境）—— 代理的自我进化

状态：Research Preview

Dreaming 是 Claude Managed Agents 的一项突破性能力。它在代理任务间隙自动运行，回顾上一会话的所有操作，识别模式、修正错误、优化工作流，并将新的记忆条目写入长期记忆。这意味着 AI 代理可以在"睡眠"期间持续改进自身表现。

实际案例：法律 AI 初创公司 Harvey 在公开发布前就已实际运行 Dreaming 功能，验证了其在生产环境中的价值。

2.2 多代理编排（Multiagent Orchestration）

状态：已发布

允许一个主代理将任务分发给多个专业子代理并行执行。子代理在共享文件系统上协同工作，结果在主代理上下文中整合。这一机制大幅提升了复杂任务的执行效率。

2.3 Outcomes（自评循环）

状态：已发布

引入独立的评估代理，按照预设标准对代理输出进行评分并提出修改建议。形成"执行—评估—修正"的闭环，确保输出质量持续达标。

2.4 自托管沙箱与 MCP 隧道

状态：自托管沙箱（Public Beta）/ MCP 隧道（Research Preview）

能力说明
自托管沙箱工具执行环境迁移至用户自控基础设施（支持 Cloudflare、Daytona、Modal、Vercel），代理编排层保留在 Anthropic 基础设施
MCP 隧道代理可访问企业私有网络内的 MCP 服务器，无需暴露公网端点，流量端到端加密

采用方：Amplitude、Clay、Rogo 等企业已在 Managed Agents 上构建基于自托管沙箱的应用。

::: section

三、开发者工具：Claude Code 动态工作流

::: summary 发布时间：2026年5月28日 | 状态：Research Preview | 可用平台：CLI、Desktop、VS Code、API、AWS Bedrock、Vertex AI、Microsoft Foundry

3.1 核心能力

动态工作流（Dynamic Workflows）让 Claude Code 能够端到端处理过去需要季度规划的复杂工程任务：

  • 并行子代理：单会话中运行数十到数百个并行子代理
  • 自动验证：结果在提交给用户前经过独立验证
  • 进度保存：中断后从断点恢复，不丢失进度
  • Ultracode 模式：新增 effort 级别，Claude 自动决定何时启用工作流

3.2 实战案例：Bun 从 Zig 到 Rust 的重写

Jarred Sumner 使用动态工作流将 Bun 从 Zig 迁移到 Rust，成果包括：

  • 约 75万行 Rust 代码
  • 99.8% 现有测试套件通过
  • 从首次提交到合并仅 11 天
  • 一个工作流映射每个结构体字段的 Rust 生命周期，另一个并行编写所有 .rs 文件

3.3 典型应用场景

  • 代码库级 Bug 搜索：并行扫描服务或仓库，独立验证每个发现
  • 大规模迁移：框架替换、API 废弃处理、跨越数千文件的语言迁移
  • 安全审计：认证检查、输入验证、不安全模式的全代码库扫描
  • 双重验证任务：独立尝试 + 对抗代理尝试破坏结果

::: section

四、定价策略：程序化与交互式用量分离

::: summary 生效日期：2026年6月15日 | 影响范围：所有付费方案（Pro、Max、Team、Enterprise）

4.1 调整背景

第三方代理工具（如 Conductor、OpenClaw）大量调用 Agent SDK（程序化用量），导致订阅额度迅速耗尽，冲击统一费率定价模式。

4.2 核心变化

变化项详情
用量分离交互式使用（Claude.ai 对话、交互式 Claude Code、Cowork）与程序化使用（Agent SDK、claude -p、GitHub Actions）切割为独立额度池
新增月度额度程序化使用获得独立月度额度（Pro: $20/月，Max 5x: $100/月，Max 20x: $200/月）
超额计费启用"额外用量"选项后，按完整 API 费率计费；未启用则暂停至下个周期
通知时间6月8日 Anthropic 发送邮件说明额度领取方式

六、企业治理与合规

6.1 自定义角色中新增管理员权限（6月2日）

Enterprise 方案的自定义角色框架扩展了管理员权限。成员可获得特定管理领域的访问权（如计费、隐私），无需被设为 Owner。这是对大型组织权限粒度管理的重要补充。

6.2 Compliance API 集成（5月21日）

Claude 新增与主流安全合规工具的 API 集成能力。IT 和安全团队可以用管理其他应用相同的方式来治理 Claude 平台及产品套件。

6.3 全球服务中断事件（6月2日）

2026年6月2日，Claude 经历了一次重大的全球服务中断，影响范围包括 Opus 4.6、Claude API 和 Claude Code CLI。Thoughtworks 的分析指出，这一事件凸显了 AI 日益成为关键基础设施的现实，企业需要将 AI 服务的弹性纳入架构设计中。

::: section

七、行业影响与展望

7.1 战略信号

  • 代理元年深化：从工具调用到自主进化（Dreaming），Anthropic 正在构建代理的完整生命周期管理
  • 开发者体验为王：动态工作流将"季度规划"级工程任务压缩到"数天完成"，重新定义软件开发效率
  • 企业安全边界重塑：自托管沙箱和 MCP 隧道回应了企业对数据安全和网络控制的刚性需求
  • 定价透明化：用量分离解决"低价格吃高用量"的公平性问题，为 Agent SDK 生态可持续发展铺路

结论

2026年6月是 Anthropic 战略落地的关键月份。从 Opus 4.8 的模型能力提升，到 Managed Agents 的"梦境"自我进化，再到 Claude Code 动态工作流的工程范式变革，以及计费模式的重大调整——这些更新共同指向一个清晰的愿景：让 AI 代理成为企业基础设施的默认组件。
