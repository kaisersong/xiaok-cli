# CEO Plan: Desktop 执行速度优化
Generated by /plan-ceo-review on 2026-05-07
Branch: master | Mode: HOLD SCOPE
Repo: kaisersong/xiaok-cli

## Problem Statement

"生成报告" 任务壁钟时间 642s（10+分钟），CLI 版显著更快（~13s）。

### Root Cause Data

```
工具执行总耗时：   0.3s  (0%)
LLM 推理总耗时：  619.6s (96%)
  ↳ Intent 协议开销（10次调用）：  127s (20%)
  ↳ 34 次 tool call 之间的模型推理： 492s (76%)
Streaming/生成：                    22.7s (4%)
```

### Why Intent Delegation is Pure Overhead

- `intent_create`: 创建意图分析（desktop 不消费 intent 数据）
- `intent_step_update` ×7: 更新步骤状态（纯追踪，无用户价值）
- `intent_stage_artifact` ×2: 记录产物（失败且无用）
- **10 次额外 LLM round-trip = 127s 纯协议开销**

### Tool Call Explosion

同一 skill（report-creator）在不同 run 中表现迥异：
- 快的：1 tool call, 13s
- 慢的：34 tool calls (14 read + 4 bash), 642s

LLM 选择展开为多轮决策而非批量执行。

### Why Desktop vs CLI Difference

数据表明同一 task 有快慢两个版本（13-14s vs 642s），差距来自 LLM 决策路径，非 desktop 特有。

## Vision

### 10x Check
如果能在 60s 内完成"生成报告"，desktop 就是可用的。当前 642s 完全不可用。

### 实现替代方案

**方案 A: 移除 Intent + 优化 Prompt（推荐）**
- 从 system prompt 和 runner 中移除 intent 工具
- 优化 system prompt 指令，引导批量执行
- 预期：642s → ~240s
- 风险：低

**方案 B: 仅移除 Intent**
- 只去掉 intent delegation，不改 prompt
- 预期：642s → ~500s（节省 127s）
- 风险：最低

**方案 C: 全量架构重构**
- 重构为 skill 预执行模式（skill 先跑完，再生成结果）
- 预期：642s → ~60s
- 风险：高 — 需要重构 task runtime
